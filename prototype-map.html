DPO Core Loss
L_DPO = -E[ log σ( β·(r_w - r_l) ) ]
r(x,y) = β · log( π_θ(y|x) / π_ref(y|x) )
β controls KL divergence penalty strength (start at 0.1).
y_w is the chosen (preferred) response, y_l is rejected.
π_ref is frozen — it provides the baseline log-probabilities.
The sigmoid σ maps the margin to [0,1], treated as a probability of correct ranking.
Sequence Log-Probability
log π(y|x) = Σ_t log p(y_t | x, y_{<t})
The model outputs a distribution over vocab at each position.
You gather the log-prob of the actual token at each position, then sum.
In PyTorch: F.cross_entropy gives negative log-prob per token → negate + sum.
def seq_logprob(logits, labels, mask): # logits: [B, T, V], labels: [B, T] log_probs = F.log_softmax(logits, dim=-1) tok_lp = log_probs.gather(2, labels.unsqueeze(-1)).squeeze(-1) return (tok_lp * mask).sum(-1) # [B]
Full DPO Loss in PyTorch
def dpo_loss(pi_logits_w, pi_logits_l, ref_logits_w, ref_logits_l, labels_w, labels_l, mask_w, mask_l, beta=0.1): lp_w = seq_logprob(pi_logits_w, labels_w, mask_w) lp_l = seq_logprob(pi_logits_l, labels_l, mask_l) ref_w = seq_logprob(ref_logits_w, labels_w, mask_w) ref_l = seq_logprob(ref_logits_l, labels_l, mask_l) # log-ratio advantage adv_w = lp_w - ref_w # [B] adv_l = lp_l - ref_l # [B] # DPO margin margin = beta * (adv_w - adv_l) loss = -F.logsigmoid(margin).mean() # diagnostics reward_gap = (adv_w - adv_l).mean().item() # should grow + return loss, reward_gap
Sector Scoring (Article → Alert)
score(a, s) = α · keyword_hits(a,s) + (1-α) · cosine_sim(emb_a, emb_s)
a = article (title + body).
s = sector definition (keywords + representative embedding).
α = blend weight, e.g. 0.4 for keyword, 0.6 for semantic.
Fire alert when score ≥ min_confidence from watched_sectors.
Start with α=1.0 (pure keyword) → add semantic matching once embeddings are set up.
Key Metrics to Track
reward_gap — E[r(x,y_w) - r(x,y_l)]. Should increase during training. If it stays near 0 or goes negative, your data has low contrast or β is too large.

KL divergence ≈ (1/β) · E[log π_θ / π_ref]. Should stay bounded — if it explodes, lower lr or raise β.

chosen_logprob increase — the policy should assign more mass to y_w over time.

Alert latency — time from SA publish to your webhook fire. Target < 60s with a polling interval of 30s.
